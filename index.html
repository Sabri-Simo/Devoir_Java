PARTIE I

Classes, Objets & Encapsulation

Q1 Déclaration et structure d'une classe +

Une classe est un modèle (blueprint) qui définit la structure et le comportement d'un objet. En Java, tout code existe à l'intérieur d'une classe. Un objet est une instance concrète de cette classe, créée avec new.

Composants principaux

Attributs (champs)
Variables qui stockent l'état d'un objet. Déclarés dans le corps de la classe.
Constructeur
Méthode spéciale du même nom que la classe, appelée lors de la création d'une instance.
Méthodes
Fonctions qui définissent le comportement de l'objet (actions, calculs).
Modificateurs d'accès
public · private · protected · (défaut)

Classe vs Objet — Analogie

La classe Voiture est le plan de construction. Chaque voiture concrète (ex. "ma Peugeot 208 rouge") est une instance (objet). Toutes partagent la même structure, mais des valeurs d'attributs différentes.

JAVA 
/**
 * Représente un livre dans une bibliothèque.
 * Domaine métier : Bibliothèque.
 */
public class Livre {

    // Attributs privés (3 types variés)
    private String  titre;
    private String  auteur;
    private int     anneePublication;
    private double  prix;
    private boolean disponible;

    /**
     * Constructeur paramétré complet.
     *
     * @param titre            Titre du livre
     * @param auteur           Auteur principal
     * @param anneePublication Année d'édition
     * @param prix             Prix en euros (≥ 0)
     */
    public Livre(String titre, String auteur,
                 int anneePublication, double prix) {
        this.titre            = titre;
        this.auteur           = auteur;
        this.anneePublication = anneePublication;
        this.prix             = prix;
        this.disponible       = true;
    }

    /**
     * Retourne une représentation textuelle du livre.
     *
     * @return chaîne décrivant l'objet
     */
    @Override
    public String toString() {
        return "[Livre] " + titre + " — " + auteur
             + " (" + anneePublication + ") | Prix: "
             + prix + "€ | Disponible: " + disponible;
    }

    // Getters basiques
    public String  getTitre()            { return titre; }
    public String  getAuteur()           { return auteur; }
    public int     getAnneePublication() { return anneePublication; }
    public double  getPrix()             { return prix; }
    public boolean isDisponible()        { return disponible; }

    // Exemple d'utilisation
    public static void main(String[] args) {
        Livre l = new Livre("L'Étranger", "Albert Camus", 1942, 8.50);
        System.out.println(l);
    }
}

Sortie : [Livre] L'Étranger — Albert Camus (1942) | Prix: 8.5€ | Disponible: true

Q2 Encapsulation et accesseurs +

L'encapsulation consiste à regrouper données (attributs) et comportements (méthodes) dans une même unité, et à contrôler l'accès extérieur via des modificateurs de visibilité.

Pourquoi private + getters/setters ?

Déclarer un attribut private empêche toute modification directe depuis l'extérieur de la classe. Les getters exposent la valeur en lecture seule ; les setters permettent une écriture contrôlée.

Un setter peut contenir une validation : refuser une valeur négative pour un prix, ou une chaîne vide pour un nom. Cela garantit l'intégrité de l'état de l'objet à tout moment.

Intégrité des données Couplage faible Maintenance facilitée Évolution sans casser l'API
JAVA 
// Ajout à la classe Livre (Question 1)

/**
 * Modifie le prix du livre avec validation.
 * @param prix Nouveau prix (doit être positif)
 */
public void setPrix(double prix) {
    if (prix < 0) {
        System.err.println(
            "[ERREUR] setPrix : le prix ne peut pas être négatif ("
            + prix + ")."
        );
        return; // on ne modifie PAS l'attribut
    }
    this.prix = prix;
}

public void setTitre(String titre) {
    if (titre == null || titre.isBlank()) {
        System.err.println("[ERREUR] setTitre : le titre ne peut pas être vide.");
        return;
    }
    this.titre = titre;
}

// ─── Programme main ───────────────────────────────────────────
public static void main(String[] args) {
    Livre l = new Livre("1984", "George Orwell", 1949, 9.99);

    // Appel avec valeur invalide → affiche l'erreur, prix inchangé
    l.setPrix(-5.0);
    System.out.println("Prix actuel : " + l.getPrix());  // 9.99

    // Titre vide → erreur
    l.setTitre("");
    System.out.println("Titre actuel : " + l.getTitre()); // 1984

    // Valeur valide → mise à jour
    l.setPrix(12.50);
    System.out.println("Nouveau prix : " + l.getPrix()); // 12.5
}

Mot-clé this()

Dans un constructeur, this(...) appelle un autre constructeur de la même classe. Cela évite la duplication de code d'initialisation. L'appel doit toujours être la première instruction du constructeur.

JAVA 
public class Rectangle {

    private double largeur;
    private double hauteur;

    /** Constructeur par défaut : carré de côté 1 */
    public Rectangle() {
        this(1.0); // → appelle Rectangle(double)
    }

    /** Constructeur à un paramètre : carré de côté donné */
    public Rectangle(double cote) {
        this(cote, cote); // → appelle Rectangle(double, double)
    }

    /** Constructeur complet : largeur × hauteur */
    public Rectangle(double largeur, double hauteur) {
        this.largeur = largeur;
        this.hauteur = hauteur;
    }

    public double surface()   { return largeur * hauteur; }
    public double perimetre() { return 2 * (largeur + hauteur); }

    @Override
    public String toString() {
        return "Rectangle(" + largeur + "×" + hauteur
             + ") | Surface="  + surface()
             + " | Périmètre=" + perimetre();
    }

    public static void main(String[] args) {
        Rectangle r1 = new Rectangle();          // 1×1
        Rectangle r2 = new Rectangle(5);         // 5×5
        Rectangle r3 = new Rectangle(4, 7);     // 4×7

        System.out.println(r1);
        System.out.println(r2);
        System.out.println(r3);
    }
}
PARTIE II

Héritage & Polymorphisme

Q4 Héritage simple +

L'héritage permet à une classe (fille / sous-classe) de réutiliser la structure et le comportement d'une autre classe (mère / super-classe), tout en les étendant ou les modifiant.

extends super() super.methode() final instanceof

Règles de visibilité héritées

Les membres public et protected de la classe mère sont hérités. Les membres private ne sont PAS accessibles directement dans la sous-classe (il faut passer par des getters). Les membres package-private sont hérités si la sous-classe est dans le même package.

Classe final

Une classe déclarée final ne peut pas être étendue (ex. String, Integer). Une méthode final ne peut pas être redéfinie dans une sous-classe.

«abstract» Animal
− nom : String
− age : int
+ Animal(nom, age)
+ manger() : void
+ parler() : String «abstract»
+ toString() : String
▲─────▲
Chien
− race : String
+ Chien(nom, age, race)
+ parler() : String
+ rapporter() : void
Chat
− indoor : boolean
+ Chat(nom, age, indoor)
+ parler() : String
+ ronronner() : void
JAVA 
// ── Classe mère abstraite ─────────────────────────────────────
public abstract class Animal {

    private String nom;
    private int    age;

    public Animal(String nom, int age) {
        this.nom = nom;
        this.age = age;
    }

    public void manger() {
        System.out.println(nom + " est en train de manger.");
    }

    public abstract String parler(); // défini dans Q5

    public String getNom() { return nom; }
    public int    getAge() { return age; }

    @Override
    public String toString() {
        return "Animal{" + nom + ", age=" + age + "}";
    }
}

// ── Classe fille Chien ────────────────────────────────────────
public class Chien extends Animal {

    private String race;

    public Chien(String nom, int age, String race) {
        super(nom, age); // appel constructeur Animal
        this.race = race;
    }

    @Override
    public String parler() { return getNom() + " dit : Wouf !"; }

    public void rapporter() {
        System.out.println(getNom() + " rapporte la balle !");
    }
}

// ── Classe fille Chat ─────────────────────────────────────────
public class Chat extends Animal {

    private boolean indoor;

    public Chat(String nom, int age, boolean indoor) {
        super(nom, age);
        this.indoor = indoor;
    }

    @Override
    public String parler() { return getNom() + " dit : Miaou..."; }

    public void ronronner() {
        System.out.println(getNom() + " ronronne paisiblement.");
    }
}

// ── main ──────────────────────────────────────────────────────
Chien rex  = new Chien("Rex",  3, "Berger Allemand");
Chat  luna = new Chat("Luna", 5, true);

rex.manger();       // hérité de Animal
rex.rapporter();    // méthode spécifique
luna.ronronner();  // méthode spécifique
Q5 Polymorphisme et classes abstraites +

Polymorphisme

Du grec "plusieurs formes", le polymorphisme permet de manipuler des objets de types différents via une référence commune. En Java, deux formes principales :

Sous-typage (runtime)
Via héritage ou interface. La méthode appelée est choisie dynamiquement selon le type réel de l'objet (dispatch dynamique).
Surcharge (compile-time)
Plusieurs méthodes de même nom avec des signatures différentes dans la même classe.
AspectClasse abstraiteInterface
Mot-cléabstract classinterface
Héritageextends (une seule)implements (plusieurs)
ÉtatPeut avoir des attributsConstantes uniquement
ConstructeurOuiNon
MéthodesAbstraites + concrètesAbstraites + default (Java 8+)
JAVA 
import java.util.*;

// parler() est déjà @Override dans Chien et Chat (voir Q4)

public class Main {
    public static void main(String[] args) {

        // Liste de type Animal → polymorphisme de sous-typage
        List<Animal> animaux = new ArrayList<>();
        animaux.add(new Chien("Rex",   3, "Labrador"));
        animaux.add(new Chat("Luna",  5, true));
        animaux.add(new Chien("Buddy", 2, "Golden Retriever"));
        animaux.add(new Chat("Milo",  1, false));

        // Dispatch dynamique : parler() résolu au runtime
        for (Animal a : animaux) {
            System.out.println(a.parler());
        }
    }
}

/* Sortie :
   Rex dit : Wouf !
   Luna dit : Miaou...
   Buddy dit : Wouf !
   Milo dit : Miaou...

PARTIE III

Interfaces & Abstraction

Q6 Interfaces +

Une interface est un contrat que les classes s'engagent à respecter. Elle définit un ensemble de méthodes (sans implémentation, sauf default) que toute classe implémentante doit fournir.

Apports de Java 8+ : méthodes default

Depuis Java 8, une interface peut contenir des méthodes default avec une implémentation concrète. Cela permet d'ajouter des fonctionnalités sans casser les classes existantes (compatibilité ascendante). Les méthodes static sont également autorisées dans les interfaces.

interface implements default static héritage multiple
JAVA 
// ── Interface métier ─────────────────────────────────────────
public interface Comparable {

    /** Retourne une valeur numérique permettant de comparer */
    double valeurComparable();

    /** Méthode default : compare this à autre */
    default boolean estPlusCherQue(Comparable autre) {
        return this.valeurComparable() > autre.valeurComparable();
    }
}

// ── Classe 1 implémentante : Livre ────────────────────────────
public class Livre implements Comparable {
    private String titre;
    private double prix;

    public Livre(String t, double p) { titre = t; prix = p; }

    @Override
    public double valeurComparable() { return prix; }

    @Override
    public String toString() { return "Livre(" + titre + " | " + prix + "€)"; }
}

// ── Classe 2 implémentante : Film ─────────────────────────────
public class Film implements Comparable {
    private String titre;
    private double note; // sur 10

    public Film(String t, double n) { titre = t; note = n; }

    @Override
    public double valeurComparable() { return note; }

    @Override
    public String toString() { return "Film(" + titre + " | ★" + note + ")"; }
}

// ── main ──────────────────────────────────────────────────────
Comparable l = new Livre("Dune", 14.90);
Comparable f = new Film("Inception", 8.7);

System.out.println(l.estPlusCherQue(f)); // true (14.9 > 8.7)
Q7 Interfaces fonctionnelles et lambdas +

Une interface fonctionnelle est une interface possédant exactement une seule méthode abstraite. L'annotation @FunctionalInterface vérifie cette contrainte à la compilation.

Interfaces fonctionnelles courantes (java.util.function)

Syntaxe lambda

Une lambda (params) -> corps est une syntaxe compacte pour implémenter une interface fonctionnelle, sans créer de classe anonyme explicite. Elle améliore grandement la lisibilité du code fonctionnel.

JAVA 
import java.util.*;
import java.util.function.*;
import java.util.stream.*;

public class BiblioDemo {
    public static void main(String[] args) {

        List<Livre> livres = List.of(
            new Livre("Dune",               14.90),
            new Livre("L'Étranger",         8.50),
            new Livre("Le Petit Prince",     6.00),
            new Livre("1984",               9.99),
            new Livre("Fondation",          12.50)
        );

        // ── 1. Tri par prix croissant (Comparator + lambda) ───
        List<Livre> tries = livres.stream()
            .sorted(Comparator.comparing(Livre::getPrix))
            .collect(Collectors.toList());

        System.out.println("=== Triés par prix ===");
        tries.forEach(l -> System.out.println(l));

        // ── 2. Filtrage : livres > 9€ (Predicate + lambda) ───
        Predicate<Livre> cher = l -> l.getPrix() > 9.0;

        List<Livre> chers = livres.stream()
            .filter(cher)
            .collect(Collectors.toList());

        System.out.println("\n=== Livres > 9€ ===");
        chers.forEach(System.out::println);

        // ── 3. Function : extraire les titres ────────────────
        Function<Livre, String> toTitre = Livre::getTitre;

        List<String> titres = livres.stream()
            .map(toTitre)
            .collect(Collectors.toList());

        System.out.println("\n=== Titres ===");
        System.out.println(titres);
    }
}

/* Sortie (ordre prix croissant) :
   Livre(Le Petit Prince | 6.0€)
   Livre(L'Étranger | 8.5€)
   Livre(1984 | 9.99€)
   Livre(Fondation | 12.5€)
   Livre(Dune | 14.9€)

   Livres > 9€ :
   Livre(Dune | 14.9€)
   Livre(1984 | 9.99€)
   Livre(Fondation | 12.5€)
*/
